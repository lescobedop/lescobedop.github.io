## ROOT CAUSE ANALYSIS

### Issue #1: URL Variant Proliferation (Crawl Budget Killer)
Google's GSC shows these pages being crawled repeatedly:
```
http://luchoescobedo.com/                    [redirects to https]
http://www.luchoescobedo.com/                [redirects to https]
https://www.luchoescobedo.com/               [redirects to https]
http://luchoescobedo.com/people/             [redirects to https]
http://luchoescobedo.com/projects/           [redirects to https]
```

**Why it happens**: GitHub Pages automatically redirects all variants to your canonical domain, but:
1. Google discovers these old variants through backlinks and internal linking
2. Google treats each as a separate page to crawl
3. Each crawl wastes time and budget on a redirect instead of content

**Impact on crawl budget**: If Google allocates 100 crawls/day to luchoescobedo.com:
- ~50 crawls go to variant URLs (http, www) → redirects (wasted)
- ~50 crawls available for actual content
- New content never gets crawled because budget exhausted on variants

### Issue #2: Archive Pages with Noindex (Secondary Drain)
Your tag, category, and year archive pages had explicit `robots: noindex, nofollow` directives:
```
/blog/tag/growth/           [noindex, but still crawled]
/blog/tag/family/           [noindex, but still crawled]
/blog/category/reflections/ [noindex, but still crawled]
/blog/2025/                 [noindex, but still crawled]
```

**Why it's a problem**:
1. Pages explicitly tell Google "don't index me" but are still crawled
2. Crawling a noindex page is wasted effort
3. These pages are linked from blog posts, so Google keeps finding them
4. Each crawl of a noindex page is budget that could go to real content

### Issue #3: New Content Starved
As a direct consequence:
```
https://luchoescobedo.com/blog/2026/Pain/               [never crawled]
https://luchoescobedo.com/institutional-intelligence/   [discovered, never crawled]
https://luchoescobedo.com/institutional-intelligence/2026/the-clarity-gap/  [never crawled]
```

These are your **business-critical institutional intelligence pieces** but they're never being crawled because the crawler is stuck on variants and archive pages.

## TECHNICAL DETAILS: What Changed

### Archive Layout Frontmatter: BEFORE
```liquid
---
layout: default
robots: noindex, nofollow
---
```

### Archive Layout Frontmatter: AFTER
```liquid
---
layout: default
sitemap: false
---
```

**Why this works**:
- `sitemap: false` is a Jekyll variable that tells `jekyll-sitemap` plugin to exclude the page
- Cleaner than `robots: noindex` because it's a structural exclusion, not a meta tag
- Allows pages to be crawled if linked but doesn't waste budget listing them in sitemap
